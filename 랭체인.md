## 랭체인

- 언어 모델에 기반하여 AI 애플리케이션을 쉽게 개발할 수 있도록 도와주는 프레임웍
- 언어 모델 API 사용을 위한 도구와 모듈을 제공하여 코드 구현을 간소화
- 언어 모델 교체도 쉽다. 언어 모델에 따른 클래스, 함수 등을 변경하지 않고 공통 모듈 사용
- 요약, 검색, 문서 생성, 질의 응답 등에 활용

### 특징
- 모듈화된 구성: 프롬프트, LLM, 메모리, 체인, 툴 등 다양한 구성 요소를 조합해 복잡한 LLM 워크플로우를 쉽게 구성할 수 있습니다.
- 언어 모델 교체 용이: OpenAI, Anthropic, Cohere, Hugging Face 등 다양한 LLM을 지원하며, 언어 모델에 따라 클래스나 함수를 변경하지 않고도 공통 인터페이스를 통해 쉽게 교체할 수 있습니다.
- 표준화된 인터페이스: langchain-core를 통해 LLM, 채팅 모델, 툴, 프롬프트 등 주요 구성 요소에 대해 일관된 API를 제공합니다.
- 체인 기반 설계: 여러 작업을 순차적으로 연결해 복잡한 작업 흐름을 구성할 수 있습니다.
- 에이전트 및 툴 통합: 외부 API, 계산기, 검색 엔진 등 다양한 툴과 연동하여 동적인 응답을 생성할 수 있습니다.

### 🧱 LangChain v0.x → v1.x 전환 (2024~2025)
- v1.0 출시: LangChain과 LangGraph가 함께 안정화되며, 구조가 대폭 개편됨.
- 마이그레이션 가이드 제공: v0.x에서 v1.x로의 전환을 위한 [공식 가이드](https://docs.langchain.com/oss/python/releases) 제공. 
- 버전 정책: LangChain은 안정성과 실험성을 구분하여 v1.x는 안정 버전, v0.x는 실험적 기능을 포함하는 구조로 운영
- v1.x 특징:

```
1) 모듈화 강화: langchain-core, langchain-community, langchain-openai 등으로 분리.
2) LangGraph 통합: 상태 기반 워크플로우를 위한 LangGraph가 공식 지원됨.
3) Pydantic 2 지원: v0.3부터 Pydantic 2로 전환되며, 더 이상 v1 브리지가 필요 없음.
4) Python 3.8 지원 종료: 2024년 10월 이후로 Python 3.8은 지원되지 않음
```

- LangGraph는 복잡한 상태 기반 워크플로우에 최적화된 그래프 구조를 제공합니다.
- 두 프레임워크 모두 LLM 기반 애플리케이션 개발을 지원하지만, 구조와 사용 목적에서 차이가 있습니다.

### 🔗 LangChain -vs- LangGraph

#### LangChain
- 선형적 체인 구조: 작업이 순차적으로 연결되어 한 방향으로 흐름이 진행됨.
- 빠른 프로토타이핑에 적합: 간단한 챗봇, 문서 요약, 툴 호출 등 단순한 LLM 애플리케이션에 유리.
- 구성 요소 조립 중심: 프롬프트, 모델, 툴 등을 체인으로 연결해 빠르게 기능 구현 가능.
- 상태 관리 제한적: 복잡한 상태 전파나 조건 분기에는 별도 구현이 필요함.

#### LangGraph
- 그래프 기반 구조: 노드와 엣지로 구성된 비선형적 흐름을 설계할 수 있음.
- 복잡한 워크플로우에 적합: 조건 분기, 반복, 병렬 처리, 에이전트 협업 등 복잡한 흐름을 자연스럽게 처리.
- 상태 기반 설계: 각 노드가 상태를 갖고, 상태에 따라 흐름이 달라짐. 중간 결과나 사용자 입력을 유연하게 관리 가능.
- 시각적 설계 가능: 저코드 방식과 시각적 인터페이스를 통해 복잡한 흐름을 쉽게 설계할 수 있음.
- 멀티에이전트 시스템에 강력: 여러 에이전트가 협력하는 구조를 명확하게 표현하고 실행 가능.
